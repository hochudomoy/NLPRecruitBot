# RecruitBot
 Multi-Agent Interview Telegram-Bot
 
**Архитектура решения:**
Система проведения технического интервью включает в себя трех AI-агентов, которые помогают друг другу в генерации ответов, имитирующих живой диалог с рекрутером. Перед началом интервью система запрашивает вводные данные о пользователе, на основе которых задается первый вопрос рекруту. Интервью завершается, когда пользователь вводит слово «стоп» или пока система не решит завершить интервью самостоятельно. Взаимодействе пользователя с системой реализовано в виде диалога с телеграм-ботом.

**AI-агент №1: InterviewerAgent – «Гигачат»**

Агент-интервьюер получает рекомендации по ведению диалога от агента №2, в соответствии с которыми формирует подходящий вопрос для пользователя и ведет с ним диалог. Также для составления вопроса агент-интервьюер использует историю диалога. Полученные ответы рекрута записываются агентом в словарь.

**AI-агент №2: ObserverAgent – «Гигачат»**

Агент-наблюдатель извлекает из словаря ответы пользователя и анализирует их. Выступая в роли критика, на основе анализа агент составляет оценку ответа кандидата: качество, off-topic, а также ошибки фактов. После оценивания агент-наблюдатель формирует рекомендации для агента-интервьюера относительно сложности следующего вопроса, а также следит, чтобы интервьюер и рекрутер не меняли заданную тему.

**AI-агент №3: SummaryAgent – «Гигачат»**

Агент-суммаризатор агрегирует данные, полученные от пользователя на протяжении всего интервью. После завершения диалога агент проводит ревью ответов и формирует отчет по структуре:
1) Decision (Grade, Hiring Recommendation, Confidence)
2) Hard Skills: Confirmed Skills и Knowledge Gaps
3) Soft Skills: Clarity, Honesty, Engagement
4) Roadmap для кандидата

**Входные данные**
Перед началом технического интервью система задает пользователю вопросы, чтобы выяснить:
1) "position" – позиция, на которую проводится интервью рекрута (например: "Backend Developer")
2) "grade" – уровень подготовки рекрута (например: "Junior")
3) "experience": - опыт работы рекрута (например: "Pet Django"; если стек технологий, которыми владеет рекрут, больше одного, то названия вводятся в строку через запятую)

**Команда для запуска**

    python main.py
